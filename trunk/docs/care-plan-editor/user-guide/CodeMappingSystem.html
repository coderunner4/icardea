Code Mapping System

In the healthcare domain, semantic encoding of the data elements is achieved by using coded terms from code systems like LOINC, ICD10 or SNOMED-CT. Even in a single EHR document, several code systems can appear at the same time. In the iCARDEA project, EHR and PHR systems may be using different code systems. Code Mapping System handles automatic mapping of coded terms from different code systems. To be able to map different code systems, HL7 Common Terminology Services (CTS) is preferred as the basis of Code Mapping API. In order to access the existing terminology servers using CTS, CTS interfaces must be mapped to the interfaces of the terminology servers. For this purpose, one of the most prominent and free terminology servers, namely Unified Medical Language System Knowledge Source Server (UMLSKS) is used. The implementation is exposed as Web Services as defined by the CTS. The Code Mapping System is composed of three modules:

  • The Code Mapping Server: This is the Web Service conforming to HL7 v3 Common Terminology Services standard.
  • The Code Mapping Database: This large MySQL Database includes the actual mapping data in the relational tables.
  • The Code Mapping Client: This Web Service client allows the other systems of the iCARDEA to reach the functionality provided by the Code Mapping Server through simple Java object calls.
The following code mapping functionalities are provided by the Code Mapping System:
  • Get a code translated from one code system to another code system: This method allows a code to be translated from source code system to a target code system.
  • Get supported mappings: This method returns the supported mappings.
  • Get supported code systems: This method returns the supported code systems.
  • Validate a given code: This method checks whether a given code is in a specific code system.

How to start

The Code Mapping System is not for human use directly. The other systems use Code Mapping System for clinical code translation. The first precondition for the Code Mapping System is the loading of UMLS data, which is about 6GB large, to a MySQL 5.1.53 database server. After the installation, the client systems can use the Code Mapping Client for code translation as follows: 
String translatedCode = tr.com.srdc.icardea.cts.CTSInvoker.translateCode(sourceCodeSystemName, code, targetCodeSystemName);

(String) sourceCodeSystemName: This parameter defines the source code system name.

(String) code: This parameter specifies the code to be translated.

(String) targetCodeSystemName: This parameter defines the target code system name.

Notes

Caution

  • If there is connection refused exception on the client side, ensure that the cts.endpoint configuration parameter points to correct url.
  • If there is connection refused exception on the server side, ensure that umls.mysql.host, umls.mysql.username and umls.mysql.password parameters point to the exact MySQL instance containing the UMLS data.

How to install

  • Install MySQL 5.1.53 database server.
  • Load the UMLS data to the server with the following command (This will take 5-6 hours): 
  • (Path/to/mysql/server)/bin/mysql metathesaurus < umls.sql
  • Install the Code Mapping Server by issuing the following command: 
  • mvn -o install -f icardea-cts-ws\pom.xml -Dmaven.test.skip=true
  • Install the Code Mapping Client by issuing the following command: 
  • mvn -o install -f icardea-ctsinvoker\pom.xml -Dmaven.test.skip=true
  • Start the tomcat server with the following command: 
  • (iCARDEA Home Directory)/bin/start_tomcat.bat

Configuration

Update the following parameters in the icardea-config\pom.xml accordingly. 
cts.endpoint		
umls.mysql.host
umls.mysql.username
umls.mysql.password

Technical Details

The only technical requirement is the installation of MySQL 5.1.53 and about 6GB of disk space.
