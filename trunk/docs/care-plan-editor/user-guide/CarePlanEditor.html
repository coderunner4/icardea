Care Plan Editor

Care Plan Definition Tool enables the definition of machine processable care plans and clinical guidelines. It provides to graphically design and personalize the medical care plans to specific patients for remote monitoring, and to bind the clinical guideline steps with remote monitoring CIED data, EHRs and PHRs of the patient.

How to start

When the basic setup configuration is done, Care Plan Editor is ready to run at http://localhost:8080/icardea_careplaneditor/flex-client/iCardea.html After filling in user name and password parts, Medical Professional is able to use to Care Plan Definition Tool by selecting from the menu on the left.

How to install

To download the source codes of the project, please go to http://www.srdc.com.tr/svn/iCardea/

After making necessary configuration changes, there are two options for installing and using iCARDEA system.

  1. This one is directly from command line and using maven commands. To start installation for the first time we need to enter following line into console (you can use setup.bat) 

    mvn install -DcreateDatabases=true -Dmaven.test.skip=true

    mvn install command (when run from icardea.home) is compiling all resources, creating WAR and JAR packages, putting this packages into local maven repository (to make them available for other projects to access it), deploying WAR files under tomcat (which is located under icardea.home/tools) and running tests.

    We used the -Dmaven.test.skip=true parameter to skip tests, since we had not started tomcat and some of our tests depends on the services provided by the WAR files deployed under tomcat. We used the -DcreateDatabases=true parameter to tell to the maven that database creation processes should be run. (This command drops all current databases).

    mvn install command does the same job whether it is called under sub-module projects. For example, when we run mvn install under icardea-careplaneditor folder, it compiles the sources of Care Plan Editor and Care Plan Monitoring Tool, creates WAR package and deploys it under tomcat (and also into maven local repository).

    If you want to skip deployment of WAR packages under tomcat, there is another parameter: -DskipDeploy=true

    After a successful installation, you need to start Tomcat using the start_tomcat.bat located under icardea.home folder. You cannot start Tomcat before installing, since it is being unpackaged into tools folder during installation. After Tomcat started, Care Plan Editor should be available at the following address. You can check them.

    Care Plan Editor: http://localhost:8080/icardea_careplaneditor/flex-client/iCardea.html

  2. As mentioned before, there is another way to install and run these tasks: "maven cli plugin" This plugin is providing a console for calling maven goals and lifecycles. To start "maven cli plugin", enter the following code into console: 

    mvn cli:execute-phase

    This will make the initial process of maven and then provide you a new console.

    maven2>

    You can use this console as a normal console and run the commands omitting the first mvn command. For example to start installation you can type:

    install -DcreateDatabases=true -Dmaven.test.skip=true

    However, if you think it is still too long, you can use the useful feature of cli plugin: "aliases". New aliases can be defined in the pom.xml in icardea.home. Several aliases have already been defined. For example to start installation with createDatabases option true and skip test option true, you can type setup to cli console. 

    setup

    setup is an alias to the install -DcreateDatabases=true -Dmaven.test.skip=true.

Configuration

After fetching the project from SVN to a local folder, first open the pom.xml file under "config" folder and make the necessary updates.

  • Set you icardea.home folder to the folder that you fetched the project.
  • Set your MySQL username and password.
  • Set max_allowed_packet to 16M in the mysql.ini configuration file.

Then set the following environment variables (you can use setenv.bat, but we recommend to set them permanently):

  • MAVEN_OPTS=-Xms256m -Xmx512m
  • CATALINA_OPTS=-Xms 256m -Xmx1024m -XX:PermSize=256m -XX:MaxPermSize=512m

If you prefer to start tomcat without using the "start_tomcat.bat", CATALINA_HOME variable should be set to {ICARDEA_HOME_PATH}/tools/apache-tomcat-6.0.30 Then open a console and go to the directory that you set as "icardea.home".

Now we are ready for installation.

Technical Details

The Adaptive Care Planner use Java(TM) SDK 1.6.0_24 (which uses Java Binary Code License Agreement) or higher and Python 2.7.1 (A very permissive, open source compliant and GPL compatible license used by the team developing the Python programming language).

Software:

  • MongoDB
  • Apache Tomcat Server 6.0.20: This server use Apache License version 2 and use 8080 port. It is operating system free and can be run in any operating system having Java 2 Standard Edition Runtime Environment (JRE) release version 5.0 or later.
  • Apache Maven 2.2.1: This build tool can be run in any operating system having Java SDK release version 1.4 or above.
  • MySQL Database 5.1.53: MySQL open source software is provided under the GPL License and support many of the operating systems. By default, it runs on port 3306.
  • Flex SDK 3.4.0: It is distributed under Mozilla Public License (MPL) version 1.1.
  • Axis2 1.5.1: This tools is also released under Apache License, Version 2.0.

Following TCP ports are required:

  • The XDS Server: 9080
  • The Update Broker: 9081 (for the web user interface), 9082 (for receiving notifications from the XDS server)
  • The Patient Index (PIX) server: 2575
  • HL7/EDF receiver: 2576
  • MongoDB : 27017 for queries and on 28017 for the web ui.
  • Observation Processor: This actor receives IHE IDCO messages and stores to the CareManagement DB. It listens post 1011.
  • MySQL Database: 3306
  • Apache Tomcat Server: 8080
