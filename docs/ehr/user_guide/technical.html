Definitions

XDS

Cross-Enterprise Document Sharing (XDS) is focused on providing a standards-based specification for managing the sharing of documents between any healthcare enterprise, ranging from a private physician office to a clinic to an acute care in-patient facility and personal health record systems.

XDS Affinity domain

An XDS Affinity Domain is a group of healthcare enterprises that have agreed to work together using a common set of policies and share a common infrastructure. An XDS Affinity Domain is equivalent to a Regional Healthcare Information Organization (RHIO) in that it defines a group of enterprises which will have an agreed method of sharing medical data. An XDS Affinity Domain does imply the exact method for sharing medical data. That method is the XDS profile which is document based and includes a single, generally accessible Document Registry which contains a document entry for every registered document. The document entry contains a defined set of metadata elements. Part of the task of setting up an XDS Affinity Domain is identifying the location of the single Document Registry (there can only be one for the XDS Affinity Domain). Another is defining a set of policy decisions which must be agreed to and abided by all participating enterprises. The policies include document format, acceptable values for some of the metadata elements of the document entries, coding systems and terminology. In the iCARDEA project the whole iCARDEA platform consists a single XDS Affinity domain.

XDS Document Registry

The Document Registry is one of the central entities needed for an XDS Affinity Domain. There can only be one per domain. It receives and manages the metadata about each available document. The Registry normally holds a URL to a repository where document actually stored but in the case of iCARDEA the Registry is “colocated” with the Repository.

XDS Document Repository

The Document Repository persists documents, registers those documents with the registry and supports retrieval of the documents. An XDS Affinity Domain can and typically will have multiple Document Repositories. PIX/PDQ Server

The PIX/PDQ Server (iCARDEA’s Patient Index) provides the server side of PIX Patient Identity Cross-Referencing and PDQ Patient Demographics Query transactions. Acting as a Patient Identifier Cross Reference Manager the PIX Manager provides cross-referencing of patient identifiers from multiple Patient Identifier Domains. It is responsible for creating, maintaining and providing lists of identifiers that are aliases of one another across different Patient Identifier Domains. Acting as a Patient Demographic Supplier the PIX manager provides ways for multiple distributed applications to query a central patient information server for a list of patients based on user-defined search criteria and retrieve a patient’s demographic information directly into the application.
