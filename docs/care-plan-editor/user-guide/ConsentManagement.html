Consent Editor and Consent Manager

The consent management in the iCARDEA project is handled through Consent Editor and Consent Manager components. The patients can define the consent rules through the Consent Editor and when the other components of the iCARDEA system try to reach the patient's data, the Consent Manager process the previously defined rules and either grants or denies the request. In addition to these components, a Consent Manager Client is also implemented.

How to start

When the basic setup configuration is done, Consent Editor is ready to run at http://localhost:8080/consenteditor/ConsentEditor/ConsentEditor.html

Notes

Caution

  • If there is connection refused exception on the client side, ensure that the consent.ws.endpoint configuration parameter points to correct url.
  • If there is connection refused exception on the server side, ensure that mysql.host, mysql.username and mysql.password parameters point to the exact MySQL instance containing the UMLS data.

How to install

  • Install MySQL 5.1.53 database server.
  • Install the Code Editor and Manager by issuing the following command: 
  • mvn -o install -f ..\icardea-consenteditor\pom.xml -Dmaven.test.skip=true
  • Install the Consent Manager Client by issuing the following command: 
  • mvn -o install -f ..\icardea-consenteditor-invoker\pom.xml -Dmaven.test.skip=true
  • Start the tomcat server with the following command: 
  • (iCARDEA Home Directory)/bin/start_tomcat.bat

Configuration

Update the following parameters in the icardea-config\pom.xml accordingly. 
consent.ws.endpoint
mysql.host
mysql.username
mysql.password

Technical Details

The only technical requirement is the installation of MySQL 5.1.53.
